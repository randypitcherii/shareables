Zero-config, except the two things only you can know

Every variable has a safe fallback — except host and warehouse http_path, which are cluster-specific. You can't tell what works out-of-the-box vs. what errors until you hit it.

Variable Env Var If Set Else Fallback Outcome
deployment_environment DBT_DEPLOYMENT_ENVIRONMENT use env value development ✅ zero-config
default_catalog DBT_DEFAULT_CATALOG use env value analytics_dev ✅ zero-config
default_schema DBT_DEFAULT_SCHEMA use env value dbt ✅ zero-config
usage_history_days DBT_USAGE_HISTORY_DAYS use env value 90 ✅ zero-config
host DBT_HOST use env value (none) ❌ REQUIRED — errors if unset
warehouse http_path DBT_HTTP_PATH use env value (none) ❌ REQUIRED — errors if unset
ci/prod client secret DBT_ENV_SECRET_DATABRICKS_CLIENT_SECRET use env value (none) 🔒 secret · masked · profiles.yml only

Happy path

git clone → set DBT_HOST + DBT_HTTP_PATH OR cp template.env .env → dbt build → ✅
